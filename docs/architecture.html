claude-python-sdk-extended

Extended Python SDK with retry logic, middleware, streaming, batch processing, and cost tracking

Python AI/ML v1.0.0

Executive Summary

An extended Python SDK built on top of the Anthropic Claude API that adds production-grade capabilities missing from the base SDK. Includes automatic retry with exponential backoff and rate limit handling, a middleware pipeline for request/response processing, streaming support with event handlers, batch processing with concurrency control, multi-turn conversation management with context windowing, structured output parsing (JSON, lists, key-value), cost tracking and usage analytics, response caching, and tool/function calling helpers. Designed as a drop-in enhancement layer for applications using the Anthropic API.

Data Flow / Request Flow

  1. Client Initialization -- Application creates a ClaudeClient with API key and optional middleware, retry, and caching configuration.
  2. Request Pipeline -- Each request passes through registered request middleware (logging, headers, transformation).
  3. Cache Check -- ResponseCache checks for cached responses to avoid redundant API calls.
  4. Retry Logic -- On API errors (rate limits, 5xx), exponential backoff with jitter retries the request.
  5. API Call -- The request is sent to the Anthropic Claude API (streaming or batch mode).
  6. Response Pipeline -- Response passes through response middleware, cost tracking records token usage, and structured output parser extracts formatted data.
  7. Conversation Context -- For multi-turn usage, the Conversation manager maintains message history with context windowing to stay within token limits.

Production Readiness Checklist

  • Set Anthropic API key via environment variable or constructor
  • Configure retry policy (max retries, backoff multiplier)
  • Set up middleware for request/response logging
  • Configure ResponseCache backend for production (Redis/disk)
  • Set batch concurrency limits appropriate for API tier
  • Configure cost tracker alerts for budget thresholds
  • Set context window size for conversation management
  • Register tool schemas in ToolRegistry
  • Test structured output parsing against expected formats
  • Monitor cost tracker metrics for usage anomalies
